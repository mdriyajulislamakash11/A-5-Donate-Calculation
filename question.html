What is the Document Vo ect Model (Dom)?

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the structure of a document as a tree of objects, where each object corresponds to a part of the document, such as an element, attribute, or piece of text.

Through the DOM, you can manipulate the document's structure, style, and content using programming languages like JavaScript. For example, you can:

1. Add or remove elements.
2. Change the content or attributes of elements.
3. Handle user interactions or events.

The DOM allows developers to dynamically update a webpage without having to reload it.

How do you select an element from the DOM?

To select an element from the DOM, you can use JavaScript methods that allow you to target specific elements by their ID, class, tag name, or other attributes. Here are some of the most common ways:

1. By ID:
You can use getElementById() to select an element with a specific ID.



2. By Class:
Use getElementsByClassName() to select elements that share the same class. This returns an HTMLCollection (like an array) of all elements with the specified class.



3. By Tag Name:
Use getElementsByTagName() to select elements by their tag name (e.g., div, p, span). This also returns an HTMLCollection.



4. By CSS Selector:
The querySelector() method allows you to select the first element that matches a CSS selector.

What is event delegation in the context of the DOM, and whay is it useful?

Event delegation is a technique in JavaScript that allows you to handle events at a higher level in the DOM rather than on individual elements. Instead of adding event listeners to each element (like buttons or links), you attach a single event listener to a parent element that can respond to events triggered by its child elements.
